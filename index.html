Semantics

All NoJS components use custom-element naming conventions nojs-{componentName}. This is because they are designed to be ready to upgraded and extended into real custom elements! By convention, all nested items must be referred to by a class. This is to prevent leaking styles down to other elements. The style is `{componentName}-{subComponent}` so, for example, a nojs-tab panel would be class tab-panel. Hidden checkboxes and radios are referred to as {componentName}-flag because they represent the 1-bit state of the component as least in my convention.

Notes and Limitations

Since we are utilizing custom element syntax and this is a prerender strategy, we need to be careful to not include rules like *:not(:defined) so the elements will appear correctly. If this rule is desired then you'll need to make sure you override it per element.
